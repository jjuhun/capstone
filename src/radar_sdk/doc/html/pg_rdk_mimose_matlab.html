MATLAB wrapper (BGT24ATR22 Doppler Radar)

1 The Mimose MATLAB wrapper

1.1 Requirements

The ATR22 (Mimose) SDK MATLAB wrapper requires an installation of MATLAB on your computer. The wrapper has been tested with MATLAB versions 2017, 2019b, and 2021b on the following platform(s).

  • Windows 10 (64bit)

1.2 Location of wrapper in release package

Extract the zip archive of the ATR22 release. The Matlab wrapper example and the wrapper functions can be found

in the extracted base directory in the subfolder MATLAB.


1.3 Help function

For further information about the application, modules and functions of the Mimose SDK MEX wrapper,

please use the help browser built into the MATLAB.

In order to call the help browser,

simply right-click on a class, function or property in the code and click "View Help" in the context menu.

Alternatively, it is possible to show the help browser by clicking on an element and pressing F1. The help and usage is also displayed on the command window when the command help precedes the function/class name.

2 Quick start guide

2.1 Start MATLAB

Start an instance of MATLAB and switch the working directory to the folder ‘MATLAB’. This is the location the Radar SDK MEX wrappers have been unzipped into. See 1.2 Location of wrapper in release package.

2.2 Attach device

Mount the ATR22 device on the RBB MCU7 and connect to the PC via USB cable.

Figure 2.1 ATR22 Mimose evaluation board

2.3 Run the example

run the example ‘MimoseWrapperExample.m’. This script provides an example performing the following steps typical for any application.

  • creates a Mimose (ATR22) device object 
    Dev = MimoseDevice();
  • provides an example of modifying the configuration 
    Dev.config.FrameConfig{1}.num_samples = 128;
Dev.config.FrameConfig{1}.selected_pulse_config_1 = true;
  • sets the configuration 
    Dev.set_config();
  • retrieves the complex data from the configured pulses and displays them. The data ‘frame’ is fetched using 
    frame = Dev.get_next_frame();

3 Advanced guide

3.1 Modifying the configuration

The configuration structure is derived from the SDK Mimose configuration which can be found in ifx_Mimose_Config_t. This configuration structure consists of 4 Pulse configurations of type ifx_Mimose_Pulse_Config_t, 2 Frame configurations of type ifx_Mimose_Frame_Config_t, 1 AFC configuration of type ifx_Mimose_AFC_Control_t, and 1 clock configuration of type ifx_Mimose_Clock_Config_t.

The further sub configuration structures can be set using the guidelines in the documentation. If the coniguration value is in the form of enums, they can be found in directory MimoseConfigOptions. an example of setting a particular configuration using enum is shown below. The user can use MATLAB autocomplete feature to view and set the options.

new_config = MimoseConfig();
new_config.PulseConfig{2}.channel = MimoseConfigOptions.ifx_Mimose_Channel_t.IFX_MIMOSE_CHANNEL_TX1_RX2;

In the example above, the ‘channel’ parameter of the second pulse configuration is set using a defined enum corresponding to it's SDK counterpart in ifx_Mimose_Channel_t.

The remaining portion of this configuration is taken from the configuration defaults coming from the SDK unless changed in this scope. Once the changes are done, the configuration can be set using

Dev.set_config(new_config);

3.3 Reading complete register list

It is helpful to read the complete register list or set registers on the device, get registers from the device when configuring aspects of the ATR22 features. An example is listed below.

  • Viewing registers in the command window
Dev.registers.view_registers();
  • Saving registers as a file
Dev.registers.view_registers('to_file',1,'filename','mimose_register_values.txt');
  • Decimal values
Dev.registers.view_registers('hex',0,'to_file',1,'filename','mimose_register_values.txt');
